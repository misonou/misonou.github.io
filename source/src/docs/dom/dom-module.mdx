# DOM module

The default export from `zeta-dom/dom` provides functionalities that handles
user interactions.

## Instance properties

### `dom.activeElement` <Badge.ReadOnly />

Gets the current active element which is readily receiving user input.

### `dom.context` <Badge.ReadOnly />

Gets the active component registered through event container.

### `dom.event` <Badge.ReadOnly />

Gets the DOM event which triggers the current event loop:

- `beforeinput`
- `click`
- `compositionstart`
- `compositionupdate`
- `compositionend`
- `contextmenu`
- `dblclick`
- `keydown`
- `keypress`
- `keyup`
- `mousedown`
- `mousemove`
- `textInput`
- `touchstart`
- `touchmove`
- `touchend`
- `wheel`

### `dom.eventSource` <Badge.ReadOnly />

Gets the type of user interaction that triggers the current event.

- `copy`
- `cut`
- `drop`
- `input`
- `keyboard`
- `mouse`
- `paste`
- `script`
- `touch`

### `dom.focusedElements` <Badge.ReadOnly />

Gets the element, and also any ancestors, which are in focus.

### `dom.metaKey` <Badge.ReadOnly />

Gets the currently pressed key modifiers, can be one of the following values:

- `alt`
- `altShift`
- `ctrl`
- `ctrlShift`
- `ctrlAlt`
- `ctrlAltShift`
- `shift`

or the empty string (`''`).

### `dom.modalElement` <Badge.ReadOnly />

Gets the current modal element.

### `dom.pressedKey` <Badge.ReadOnly /> <Badge.Version />

Gets the currently pressed key, prefixed by `dom.metaKey` when meta keys like `Ctrl` or `Shift` is also pressed.
Empty string is returned if there are no keys being pressed.

### `dom.ready` <Badge.ReadOnly />

Gets a promise which is resolved when DOM is ready.

### `dom.root` <Badge.ReadOnly />

Gets the root element of the document, usually the `<html>` element.
