# DateField class

The `DateField` class are not supposed to be instantiate manually. Instead it is used to
augment extra functionalities when used with [`useFormField`](:).

## Static methods

### `DateField.getDate()` <Badge.Version />

```javascript
getDate(str)
getDate(str, base)
```

Returns a date relative to another date.

Supports relative dates in the following format:
- A `+` or `-` sign, then number and unit (`y`, `m`, `w` and `d` for years, months, weeks and days), e.g. `+1d` 
or `-1y`.
- Multiple number and unit pairs can be specified, e.g. `+1y1d` or `-1y1d` which is evaluated from left to right(order matters because the months unit may result in different date due to various days in different month).    
- Different signs can also be specified for each number and unit pair, e.g. `+1y-1d`.

### Parameters

<dl>
    #### `str`
    A string specifying the relative date.

    #### `base` <Badge.Optional />
    An optional base date which calculation is based on. Default to today.

</dl>

### Return value

A string in standard format (YYYY-MM-DD), or an empty string if the base date is invalid.

### `DateField.toDateObject()`

```javascript
toDateObject(str)
```

Converts a given string to its representated date, truncating time component to midnight in local time.

If the string is in standard format (YYYY-MM-DD), the resulting date object will always have the represented date in local time.   
Otherwise it is up to browser to determine the original timezone and perform timezone conversion if necessary.

#### Parameters

<dl>
    #### `str`
    Any string representation for a date.

</dl>

#### Return value

A `Date` object or `null` if the input string was invalid.

### `DateField.toDateString()`

```javascript
toDateString(date)
```

Converts a given date to its string representation.

#### Parameters

<dl>
    #### `date`
    A `Date` object.

</dl>

### Return value

A string in standard format (YYYY-MM-DD), or an empty string if the date is invalid.

## DateFieldProps interface

<MemberList
    i="DateFieldProps"
    extends={['FormFieldProps']}
    ip={['min', 'max', 'formatDisplay']}
/>

### `DateFieldProps.formatDisplay`

Formats display text for a given date.

### `DateFieldProps.max`

Specifies the latest date allowed.
It accepts `Date` object, timestamp in milliseconds, any parsable date string, or relative date from today.     

> For relative date format, see [`DateField.getDate()`](:).

### `DateFieldProps.min`

Specifies the earliest date allowed.
It accepts `Date` object, timestamp in milliseconds, any parsable date string, or relative date from today.     

> For relative date format, see [`DateField.getDate()`](:).

## DateFieldState interface

<MemberList
    i="DateFieldState"
    extends="FormFieldState"
    ip={['min', 'max', 'displayText']}
/>

### `DateFieldState.displayText` <Badge.ReadOnly />

Formatted text returned by `formatDisplay` or the standard format (YYYY-MM-DD) by default.

### `DateFieldState.max` <Badge.ReadOnly />

Latest allowed date in the standard format (YYYY-MM-DD),
or an empty string if `max` is not specified or is invalid.

### `DateFieldState.min` <Badge.ReadOnly />

Earliest allowed date in the standard format (YYYY-MM-DD),
or an empty string if `min` is not specified or is invalid.
